- 데이터: 원본 232장 + 증강 → 외부 데이터 추가 수집
- 최종 성과: mAP@[0.75:0.95] = **0.99524**

## 2) 데이터 현황 및 문제 진단

원본 데이터의 구조와 품질 이슈를 파악하는 단계입니다.

### 원본 데이터 구조

데이터는 COCO 형식 JSON(`annotation_id`, `image_id`, `category_id` 등)으로 제공되었으며, 알약 1개당 JSON 1개로 쪼개져 있어 이미지 단위 병합이 필요한 상태였습니다. 원본 규모는 train 232장, test 842장입니다.

### 발견된 문제

1. **train/test 비율 불균형**: train 232장 대비 test 842장으로 학습 데이터가 크게 부족
2. **bbox 소실**: 일부 이미지에서 annotation이 누락
3. **bbox 유격**: bbox가 알약 경계와 정확히 맞지 않는 케이스 존재
4. **중복 bbox**: 동일 알약에 대해 bbox가 중복 생성
5. **알약별 데이터 분포 편차**: 클래스 간 샘플 수 차이가 큼

### 외부 데이터 검수

추가된 학습용 외부 데이터에서 bbox annotation 오류가 있는 케이스를 발견하였고, 정성적 평가를 진행하여 오류를 탐지한 뒤 제외할 이미지 리스트(총 208개)를 작성하였습니다. 초반에는 코딩을 통해 제외 리스트를 자동 선별하기를 수차례 시도하였으나 의도한 대로 진행되지 않아, 2만여 개 이미지를 직접 검수하였습니다.

## 3) 데이터 해결 및 최종 구성

각 문제에 대한 판단 근거와 해결 방법, 그리고 최종 학습 데이터 구성을 정리합니다.

### 문제별 해결

- **bbox 소실 / bbox 유격**: 모델 필터링으로 걸러지지 않는 데이터는 전수 검수 후 처리
- **데이터 불균형**: 100개 미만 클래스에 한하여 300개 규모 증강 적용 및 Kaggle 추가 데이터 적용
- **중복 bbox**: 중복 bbox 처리 코드를 적용하여 해당 이미지 데이터 자체를 삭제
- **JSON 병합**: 이미지 1개당 JSON 4개 → 이미지 1개당 라벨 1개로 통합, COCO 형식 → YOLO 형식 변환

### 병합 결과

- `merged_images`: 232 / `merged_annotations`: 762 (이미지당 평균 3.284개, 최대 4개)
- 분포: 4개 74장 / 3개 150장 / 2개 8장
- `merged_categories(nc)`: 56 / invalid annotation 1개 자동 제외

### YOLO 변환 및 스플릿

- train/val = 209/23 (seed=42 고정)
- 라벨 txt 232개 생성 완료, 이미지 누락 0

### 최종 데이터 구성

원본 데이터만으로는 학습 데이터가 부족하여 외부 데이터를 추가 수집하였고, annotation 오류 208건을 제외한 정제 데이터를 최종 학습에 사용하였습니다. 이미지 해상도는 원본 데이터와 통일하였습니다.

## 4) 파이프라인 및 재현성 설계

전체 실행 흐름과 프로젝트 구조를 통한 재현성 확보 방법을 정리합니다.

### 전체 파이프라인

```
데이터 수집 ┌   EDA/분석 ─ 전처리(병합/정제)  ┐
          ├ Baseline 개발 ─ Baseline 개선  ├ 추론 ─ 후처리 ─ Kaggle 제출
          └           모델 학습            ┘
```

### 프로젝트 디렉터리 구조

- `scripts/`: 순서대로 실행 시 전체 파이프라인 재현
- `configs/`: YAML 기반 실험 설정 관리 (코드 변경 없이 실험 조건 변경)
- `src/`: 재사용 가능한 라이브러리 코드 분리
- `runs/`: 실험별 결과 자동 저장 (가중치/로그/설정 등)
- `artifacts/`: 최종 산출물 (베스트 모델, 제출용 CSV)

### 설계 철학

- **원커맨드 재현**: `run_pipeline.sh` 하나로 전체 실행
- **실험 추적**: `_registry.csv`를 통한 실험 이력 관리
- **코드·설정 분리**: 실험 조건은 YAML, 로직은 Python

## 5) 실험 흐름 및 모델 개선

Baseline 수립부터 최종 모델까지의 실험 흐름을 정리합니다.

### Baseline 설정 및 결과

- **모델**: YOLOv8s (Pretrained, COCO) / **Backbone**: CSPDarknet
- **데이터 조건**: 전체 데이터, Mosaic, RandomFlip
- **하이퍼파라미터**: Epochs 100, imgsz 640, Batch auto(-1), Seed 42, Patience 20, optimizer/lr Ultralytics 기본
- **결과**: mAP@0.5 = 0.98037, mAP@[0.75:0.95] = **0.96138**

Baseline 실험의 핵심은 데이터 구조 문제로 학습/제출이 깨지는 리스크를 제거하는 것이었습니다. COCO annotation을 `file_name` 기준으로 병합하고, bbox/참조 무결성 검증 후 YOLO 포맷으로 변환하였으며, 제출 시 이미지당 top-4(score 기준)를 강제 적용하여 Kaggle 정상 채점을 확인하였습니다.

### Baseline 한계 분석

- test 추론에서 raw 예측 박스가 이미지당 평균 ~20개로 과다하여 후처리 개선 여지 존재
- Recall이 Precision 대비 낮은 편(0.88 vs 0.82 근처)으로, 경계가 흐리거나 작은 알약에서 놓치는 케이스 확인

### 모델 개선 테이블

| No | 모델 | 비고 | mAP@[0.75:0.95] |
|---|---|---|---|
| 1 | YOLOv8n | — | 0.90052 |
| 2 | YOLOv8n | 초기 Baseline Model | 0.96138 |
| 3 | YOLOv8m | Baseline Model 개선 | 0.96859 |
| 4 | YOLOv8s | 데이터 추가 및 후처리 전략(top-4) 도입 | 0.98821 |
| 5 | YOLOv8m | BBOX 정밀도 및 이원화 전략 | **0.99524** |

### 점수 추이 및 핵심 변경점

1. **0.96138 → 0.96859**: 증강 제거. 통제된 상황의 데이터이기에 증강이 오히려 악영향을 주었고, 이미지 플립 역시 불필요하였음
2. **0.96859 → 0.98821**: 외부 데이터 추가, 이미지 해상도 원본 통일, top-4 전략 적용, min_conf=0.15 이하 제거
3. **0.98821 → 0.99524**: 학습 데이터의 정합성 향상 (annotation 오류 제외 등)

## 6) 최종 성과

프로젝트의 정량적 결과를 정리합니다.

- 최종 mAP@[0.75:0.95]: **0.99524**
- 초기 대비 향상: 0.90052 → 0.99524 (**+10.52%p**)
- Baseline 대비 향상: 0.96138 → 0.99524 (**+3.52%p**)
- 핵심 기여 요인:
  1. 추가 데이터 수집 및 정제 (2만여 개 직접 검수 포함)
  2. 모델 스케일 선택 (v8n → v8m → v8s → v8m)
  3. 후처리 전략 (top-4, conf threshold 튜닝)

## 7) 회고

프로젝트를 마무리하며 성장 포인트를 정리합니다.

### 잘한 점

- 데이터 검증 파이프라인 구축으로 실험의 변수 통제가 우수하였음
- 체계적 디렉터리 구조로 실험 재현성을 확보하였음
- 일 2회(10시, 16시) 짧은 미팅을 통해 팀원 상태 공유 및 도움 요청이 원활하였음

### 아쉬운 점

- **실험 자동화 파이프라인 미구축**
  - 실험 실행/평가/정리 과정의 수작업 비중이 높아 반복 비용이 컸음
  - 설정 변경 후 결과 비교까지의 리드타임이 길어짐
- **앙상블 실험 미시도**
  - 단일 모델 중심으로만 최적화를 진행해 성능 상한 탐색이 제한됨
  - 모델 간 상호 보완 효과(WBF/NMS 기반 결합)를 검증하지 못함
- **데이터 불균형 예외 처리 부족**
  - 소수 클래스/경계 케이스에서 예외 상황을 충분히 방어하지 못함
  - 일부 증강/후처리 단계에서 edge case 대응 테스트가 부족했음

### 다음 프로젝트 적용 사항

- **실험 자동화 우선 구축**
  - `run -> evaluate -> report`를 원클릭으로 연결하는 자동화 스크립트 도입
  - 실험 메타데이터 자동 기록으로 비교/회고 비용 절감
- **앙상블을 필수 실험 단계로 편성**
  - 단일 모델 수렴 후 `v8/v11 + WBF/NMS` 조합을 표준 실험으로 고정
  - 제출 직전 단일모델 vs 앙상블 성능을 체크리스트로 의무 비교
- **불균형/예외 처리 테스트 강화**
  - 클래스별 최소 샘플 기준과 bbox 경계 검증을 사전 테스트로 고정
  - 소수 클래스 중심의 실패 케이스 회귀 테스트를 CI 항목으로 반영
